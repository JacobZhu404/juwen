# 聚问 (JùWèn)

一次提问，同时发给多个国产大模型网站，并排对比答案。

不依赖任何官方 API —— 通过 Playwright 驱动一个常驻的 Chrome 浏览器，直接操作各家的网页对话框。登录态保存在本地，登录一次即可长期复用。

## 快速开始

```bash
cd /Users/jacob/personal/llm-aggregator
npm install            # 首次会自动安装 Playwright 的 Chromium
PORT=5178 npm start
```

然后打开 http://localhost:5178

### 首次登录

每个模型在顶部都有一个状态徽标。点击「登录」按钮会弹出对应站点的浏览器窗口，手动登录一次即可。登录态保存在本地的 `.browser-profile/`，之后无需重复登录（个别站点会话过期后需重新登录，见下文）。

## 使用

- 在底部输入框输入问题。
- **回车 = 换行，Shift + 回车 = 发送**（也可以点「发送」按钮）。发送后输入框自动清空。
- 每个模型一栏，答案实时流式显示。
- 用顶部每个模型的复选框可临时启用/停用某个模型（选择会被记住）。
- **复制**：每栏顶部有单独的「复制」按钮（复制该模型最新回答）；顶部「复制全部」会把所有已启用模型的最新回答合并复制。

## 多轮对话与历史

- 每一栏都是一个**可滚动的对话记录**：每次提问都会在每个模型下追加一轮「问题 + 回答」，往上滚就能看到历史。
- 提问默认**沿用各站点当前打开的对话**继续追问（多轮上下文有效），无需手动在每个模型页面预先创建专门的会话。流式逻辑用提问前的最后一条回答作为基线，避免把上一轮旧答案再吐出来。
- 历史记录保存在浏览器 localStorage，刷新页面不会丢（往上滚即可看历史）。
- 每一轮回答下方有两个小按钮：
  - 「**刷新**」：不重新提问，只重新抓取该模型当前网页上的答案来更新这一格。适合"网站其实已经答完，但我们这边流式抓取提前停了"的情况。
  - 「**重试**」：在该模型上重新发送本问题、单独重答这一格（如豆包人机验证、文心登录失效，处理完后点它补答）。两者都只作用于该模型，不影响其他列。
- 顶部「**新对话**」会清空历史，并让所有已启用模型开启一段全新对话（清掉上下文）。

## 注意事项

- **文心一言**的登录态过期时，登录墙只在“发送”那一刻才弹出（页面被动检测仍显示已登录）。如果文心报「登录已失效」，点它的「登录」按钮重新登录即可。
- 各家网站的 DOM/class 名经常变化且常被混淆。如果某个模型抓不到答案或抓错内容，需要到 `adapters/<模型>.js` 里调 `selectors`。
- 调选择器的利器：`GET /api/inspect/:id`，支持 `?ask=问题&wait=毫秒` 实际发送一次并 dump DOM，`?sel=A||B||C` 探测候选选择器命中情况，`?find=某段答案文字` 反查真正承载答案的节点。

## 结构

```
server.js          Express + WebSocket，/api/* 接口，并发调度各模型
browser.js         常驻 Chrome（launchPersistentContext），每个模型一个 tab
adapters/
  base.js          通用的登录检测 / 发送 / 流式抓取逻辑（配置驱动）
  index.js         模型注册表
  <模型>.js        各站点的 selectors + URL
public/            前端（原生 HTML/CSS/JS）
.browser-profile/  本地登录态（不要提交、不要分享）
```

新增一个模型：在 `adapters/` 下照已有文件写一个 `new Adapter({...})`，再加进 `adapters/index.js` 的 `adapters` 数组。
